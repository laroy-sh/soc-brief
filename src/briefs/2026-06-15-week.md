## Act by

- **15 Jun 2026** — Older Sentinel **repositories (content-as-code) Source Control API** versions become unsupported after today. CI/CD tooling that creates or manages repo connections must move to API version 2025-09-01, 2025-06-01, or 2025-07-01-preview; existing connections keep operating. If your team deploys analytics rules and playbooks from GitHub or Azure DevOps, confirm the pipeline is on a supported API version. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/sentinel/whats-new))
- **1 Jul 2026** — **Account-entity naming standardization** takes effect (heads-up, ~2 weeks out). Sentinel resolves Account Name by precedence and defaults to the UPN prefix (the part before "@"). Analytics rules, automation rules, playbooks, workbooks, and hunting queries that rely on strict full-UPN matching break unless updated — regardless of whether you have migrated to the Defender portal. Audit your queries for hard-coded full-UPN account matches now. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/sentinel/whats-new))
- **1 Jul 2026** — **Unified RBAC migration** groundwork before the Defender-portal cutover (heads-up). Legacy Sentinel Azure roles do not carry into the Defender portal, so plan URBAC migration to avoid permission gaps; Workspace Manager deprecation pushes MSSPs toward CI/CD via GitHub or Azure DevOps. ([Microsoft Learn](https://learn.microsoft.com/en-us/unified-secops/whats-new))

## What changed

**June Patch Tuesday** landed on 9 June 2026 and was Microsoft's largest ever, addressing more than 200 CVEs with roughly three dozen rated Critical. Microsoft did not flag any of the June CVEs as exploited in the wild at release, but several were publicly disclosed beforehand — including a Windows Collaborative Translation Framework (CTFMON) elevation-of-privilege bug (CVE-2026-45586), a Windows BitLocker security-feature bypass, and an HTTP.sys denial-of-service flaw. On-premises Exchange also received a June security update that belongs near the top of the patch queue. For a Microsoft SOC the practical work is confirming the Defender-delivered updates roll out through MDE and watching endpoint patch compliance in vulnerability management, with internet-facing HTTP.sys and Exchange hosts prioritized. ([MSRC Security Update Guide](https://msrc.microsoft.com/update-guide/releaseNote/2026-Jun))

## Worth knowing

This was a genuinely quiet week for first-party feature news. The Sentinel, Defender XDR, and Entra blogs published nothing dated between 9 and 15 June — the June feature wave (the Agent Identities Asset Connector for Sentinel and related data-lake and identity items) begins landing from 16 June, so those announcements fall into the following week's brief rather than this one. Nothing this window requires an analyst to change a detection or playbook beyond the deadline items above. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/sentinel/whats-new))
