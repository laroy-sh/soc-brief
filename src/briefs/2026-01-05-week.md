## Act by

- **1 Jul 2026** — Microsoft Sentinel standardizes how account entities are named in incidents and alerts. The `Account` value now resolves by priority: UPN prefix (the part before `@`), then Name, then Display Name. This can break analytics rules, automation rules, playbooks, workbooks, and hunting queries that key off the old value. Audit your detections and automation now, and rewrite lookups to be precedence-aware, e.g. `coalesce(Account.UPNprefix, Account.Name, Account.DisplayName)`, testing in a non-production workspace first. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/sentinel/whats-new))

## Worth knowing

The Defender-portal migration remains the background theme heading into 2026. Microsoft Sentinel is generally available in the unified Microsoft Defender portal (including for customers without Defender XDR or an E5 license), and the Azure-portal Sentinel experience is on a retirement path — so if your team still lives in the Azure portal, use the quiet period to plan the move. A useful practical read from just before this window is Microsoft's guidance on how Sentinel and Defender XDR permissions map together in the unified portal, which is the most common source of "why can't I see this incident" confusion after onboarding. ([Microsoft Sentinel blog](https://techcommunity.microsoft.com/blog/microsoftsentinelblog/managing-microsoft-sentinel-and-microsoft-defender-xdr-permissions-in-microsoft-/4480583))

No Patch Tuesday falls in this window. The January 2026 security update release lands on the second Tuesday, 13 January, so expect the monthly Microsoft CVE batch — and any actively-exploited items — next week rather than this one. ([MSRC Security Update Guide](https://msrc.microsoft.com/update-guide))
