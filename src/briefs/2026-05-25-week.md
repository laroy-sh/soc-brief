## What changed

**Cloud security reporting in the Microsoft Defender portal** entered public preview, adding built-in and custom reports for Defender for Cloud data under a new **Reporting > Cloud** tab. Analysts get predefined views such as the **CNAPP Executive Summary** (threat detection, secure score trends, vulnerability management, recommendations, investigation activity, and regulatory compliance) and **Cloud Posture**, and can duplicate and customize sections, export to PDF, and control access with Private, Tenant-level, or Public visibility. For a SOC, this puts cloud posture and CNAPP signals into shareable reports without pushing data into a separate BI tool. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/defender-for-cloud/release-notes))

**Defender for Cloud vulnerability scanning now covers Docker Hardened container images** in preview, using Microsoft Defender Vulnerability Management to identify vulnerabilities so teams can confirm they ship the most secure builds. The rollout is gradual and requires no user action, though scanning more image types may increase cost. For analysts triaging container findings, it closes a blind spot in registries that rely on hardened base images. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/defender-for-cloud/release-notes))

**Microsoft Purview Data Security Posture Management (DSPM)** reached general availability with a reworked experience that unifies discovery, protection, and remediation in a single workflow. Goal-oriented flows, deeper remediation, expanded reporting, and third-party visibility let teams discover sensitive data, assess risk, and take action at scale from the same place. For a SOC, DSPM is where data-exposure risk gets surfaced and triaged before it becomes an exfiltration incident. ([Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/21/whats-new-in-microsoft-security-may-2026/))

**Purview Data Security Investigations** added optical character recognition (OCR) and custom examinations. OCR extracts text from images, bringing previously unreadable visual content into scope for AI-powered deep content analysis, and custom examinations let investigators define their own analysis beyond the built-in credential, risk, and PII checks. This gives an analyst running a data-incident investigation broader coverage and more tailored scoping. ([Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/21/whats-new-in-microsoft-security-may-2026/))

**Microsoft Entra ID Account recovery** provides an advanced recovery path for users who have lost access to all their registered authentication methods. Unlike a traditional password reset, it focuses on identity verification and trust re-establishment before new authentication methods are issued. For a SOC, a verification-gated recovery flow narrows the help-desk social-engineering path attackers use to seize accounts and makes recovery after a confirmed compromise safer. ([Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/21/whats-new-in-microsoft-security-may-2026/))

**Microsoft Purview visibility extended to Anthropic's Claude** through a new Claude Compliance API, letting security and compliance teams detect and investigate Claude Enterprise usage alongside other cloud apps. Purview surfaces Claude interactions and audit-log signals for centralized oversight of sensitive data moving through the AI estate. For analysts, it brings a widely used third-party AI tool into the same audit and investigation surface as the rest of the environment. ([Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/21/whats-new-in-microsoft-security-may-2026/))

## Worth knowing

In-window activity was the monthly Microsoft Security roundup plus day-to-day Defender for Cloud release notes, so expect the next few briefs to carry the Build wave.
